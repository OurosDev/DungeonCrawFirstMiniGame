# v0.10 — Grimoire de combat et ciblage des soins

Date de préparation : 2026-06-19

## Objectif

`v0.10` enrichit le combat sans changer la boucle générale du jeu : le grimoire obtient une version de combat propre au héros actif, les sorts de soin en combat peuvent cibler directement un héros via les cadres latéraux, et le canal de messages Combat devient plus lisible.

Cette version prépare une future base de sorts plus complexe, sans ajouter d'étage, sans consommables, sans journal de quête et sans changement volontaire du format de sauvegarde.

## Ajouts principaux

### Grimoire de combat

- Ajout d'un bouton `Grimoire` pendant le combat pour les héros ayant des sorts utilisables.
- Le grimoire de combat est propre au héros actif : un héros ne peut préparer que ses propres sorts pendant son propre tour.
- Les sorts actifs sont temporaires et sont réinitialisés à l'entrée de chaque combat.
- Le système ne sauvegarde pas encore de préférence de sort actif.

### Sorts actifs temporaires

- À l'entrée en combat, le sort offensif actif est réinitialisé sur le sort de magie de base disponible.
- À l'entrée en combat, le sort de soin actif est réinitialisé sur le sort de soin de base disponible.
- `Magie` lance directement le sort offensif actif sans ouvrir de menu intermédiaire.
- `Soin` lance directement le sort de soin actif et passe immédiatement en sélection de cible par cadres.

### Règles de tour du grimoire de combat

```text
Sélectionner un sort différent du sort actif :
-> le sort actif change ;
-> l'action du héros est consommée ;
-> le tour avance normalement.

Sélectionner le sort déjà actif :
-> le grimoire se ferme ;
-> le héros garde son tour.

Retour / E / Échap / clic retour :
-> le grimoire se ferme ;
-> le héros garde son tour.
```

Cette règle évite de pénaliser le joueur lorsqu'il consulte le grimoire pour vérifier le sort actif déjà préparé.

### Ciblage des soins en combat

- Le bouton `Soin` ne masque pas le combat par un panneau central.
- La sélection passe directement aux cadres de personnages latéraux.
- La cible active reçoit une bordure verte.
- La barre de PV de la cible affiche une prévisualisation du soin attendu.
- La barre de PM du lanceur affiche une prévisualisation du coût du sort.
- La validation fonctionne à la souris, avec `A` / `Entrée`, et la navigation reste compatible flèches + `ZQSD`.
- L'annulation fonctionne avec `E` / `Échap`.

### Canal Combat plus lisible

- Les lignes de dégâts joueur vers ennemi conservent la couleur combat actuelle.
- Les lignes de dégâts ennemi vers joueur sont colorées en rouge.
- Les lignes de soin sont colorées en vert.
- La coloration se fait ligne par ligne afin de gérer correctement les entrées de combat multi-lignes.

## Notes de design

Le grimoire de combat sert à préparer ou vérifier le sort actif du héros actif. Il ne remplace pas les boutons `Magie` et `Soin` : ces boutons restent des actions directes afin de préserver le rythme du combat.

À terme, quand il y aura plus de sorts, le grimoire hors combat pourra devenir individuel par héros et permettre de choisir les sorts actifs avant le combat. Les sorts utilisables seulement en combat pourraient alors être visibles mais grisés, tout en restant cochables comme sorts actifs.

## Compatibilité sauvegarde

Cette version ne change pas volontairement le format de sauvegarde.

Les sorts actifs de combat sont temporaires et réinitialisés au début de chaque combat. Ils ne sont pas persistés. Les changements de PV/PM pendant le combat restent déjà couverts par la sauvegarde normale des héros après sortie du combat.
