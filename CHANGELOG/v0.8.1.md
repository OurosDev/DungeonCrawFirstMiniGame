# v0.8.1 — Stabilisation playtest et scaling fenêtre

Date : 2026-06-18  
Statut : correctif de stabilisation  
Base : `v0.8`

---

## Résumé

`v0.8.1` est une version de stabilisation courte après la première session de playtest externe de `v0.8`.

Elle ne modifie pas le gameplay. Elle fixe surtout deux points de confort/stabilité :

- validation de la procédure de build Windows en `Compatibility / OpenGL` pour les playtests ;
- correction du scaling de fenêtre avec une interface lisible en résolution élevée.

---

## Affichage / scaling

Le projet utilise désormais un stretch global plus adapté à l’interface actuelle :

```ini
window/stretch/mode="canvas_items"
window/stretch/aspect="keep"
window/stretch/scale=1.0
window/stretch/scale_mode="fractional"
```

Ce réglage a été retenu après comparaison avec `viewport`.

Résultat validé :

- la fenêtre peut être agrandie sans casser les proportions générales ;
- l’interface reste adaptée ;
- les textes restent plus lisibles qu’avec le rendu `viewport` étiré ;
- aucun script n’a été nécessaire pour ce correctif.

---

## Renderer / compatibilité playtest

Le projet est aligné sur le renderer `Compatibility / OpenGL` :

```ini
config/features=PackedStringArray("4.6", "GL Compatibility")
renderer/rendering_method="gl_compatibility"
```

Décision de workflow :

- les builds Windows de playtest doivent utiliser `Compatibility / OpenGL` par défaut ;
- les builds `.exe`, `.pck` et `.zip` restent locales et ne sont pas poussées dans le repo ;
- les logs complets restent hors repo ;
- les documents de playtest utilisent uniquement des résumés nettoyés/réécrits des logs si nécessaire.

---

## Incident playtest documenté

Lors du `PLAYTEST_01 — v0.8`, le testeur a rencontré des crashs natifs Windows avec les builds utilisant le renderer moderne. La build `Compatibility / OpenGL` a réglé le problème sur sa configuration.

L’incident est considéré comme résolu par procédure d’export, sans correction gameplay sur les coffres ou la sauvegarde.
