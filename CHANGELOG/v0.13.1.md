# v0.13.1 — Correctifs UI, stèles de sort et flags de build

Date de préparation : 2026-06-22

## Objectif

`v0.13.1` est une release de stabilisation après `v0.13 — Magicka`.

Elle regroupe les correctifs locaux validés autour :

- du flag de téléportation de développement ;
- de la représentation visuelle des sorts découverts dans le donjon ;
- de la fermeture propre du menu d'aventure ;
- de la cohérence des boutons de fermeture ;
- du polish du menu inventaire.

## Build / développement

Nouveau fichier :

```text
scripts/core/BuildFlags.gd
```

Il centralise le flag :

```gdscript
const DEV_TELEPORT_ENABLED: bool = false
```

Comportement :

```text
true  = outil de téléportation disponible pour test local
false = outil de téléportation masqué et bloqué pour build playtest/exécutable
```

La téléportation est bloquée à deux niveaux :

```text
- interface : bouton T masqué ;
- donjon : debug_teleport_to_cell() refuse l'action si le flag est false.
```

## Stèles de sort

Nouveau symbole de carte :

```text
S = Stèle de sort
```

Changements :

```text
- Étage 1 : la découverte spell_ice_shard est représentée par une case S.
- Étage 2 : la découverte spell_group_heal en x21 y8 est représentée par une case S.
- L'automap / carte agrandie affiche S avec une couleur magique.
- DungeonRenderer.gd crée un modèle 3D de stèle magique pour les cases S.
- La stèle reste visible après découverte du sort.
```

Orientation :

```text
- la stèle regarde d'abord une case ".";
- sinon elle regarde une case praticable non-mur ;
- elle évite donc de faire face à un mur quand un chemin adjacent existe.
```

## Menu d'aventure

Correction du bug suivant :

```text
Fermer le menu d'aventure avec Échap pouvait laisser les boutons d'exploration invisibles ou inactifs jusqu'au prochain déplacement clavier.
```

Correction :

```text
- fermeture propre via close_requested ;
- restauration explicite des commandes d'exploration ;
- retour immédiat des boutons Avancer / Reculer / Gauche / Droite / Carte / Menu.
```

Ajout :

```text
- bouton X en haut à droite du menu d'aventure ;
- le X et Échap utilisent la même logique de fermeture.
```

## Menu inventaire

Changements :

```text
- suppression du cadre parasite sous / autour de Retour menu ;
- suppression de l'indication superflue "Échap : retour au jeu" ;
- réajustement vertical des cadres ;
- agrandissement contrôlé de la zone principale d'inventaire ;
- bouton Retour menu conservé et repositionné correctement dans le cadre principal.
```

## Carte agrandie

Changement de cohérence UI :

```text
- suppression du bouton Retour dans la carte agrandie ;
- ajout d'un bouton X en haut à droite ;
- le X ferme la carte et restaure les commandes d'exploration.
```
