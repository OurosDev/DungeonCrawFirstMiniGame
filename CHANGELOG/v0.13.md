# v0.13 — Magicka : progression magique, sorts actifs et poison

Date de préparation : 2026-06-22

## Objectif

`v0.13` transforme la magie en véritable axe de progression.

La version ajoute des sorts plus avancés, une préparation hors combat des sorts actifs, un soin de groupe et un premier système de statut temporaire avec le poison.

## Gameplay magique

### Mage

`Éclat de givre` est rééquilibré :

```text
coût : 10 PM
dégâts : 12-24
prérequis : Mage niveau 2 + découverte spell_ice_shard
```

Il devient un sort supérieur clair : plus coûteux qu'`Étincelle`, mais environ 1,5 fois plus puissant.

Nouveau sort :

```text
Poison
classe : Mage
niveau requis : 5
coût : 10 PM
cible : 1 monstre
effet : applique Poison
```

Le poison inflige 5 à 10 % des PV max du monstre à la fin de son tour.

Règle de dissipation :

```text
après le 1er tick : 1 chance sur 6
après le 2e tick : 1 chance sur 4
après le 3e tick : 1 chance sur 2
après le 4e tick : disparition automatique
```

Le boss gardien est immunisé au poison pour préserver l'équilibrage actuel.

### Prêtre

Nouveau sort de progression par niveau :

```text
Soin renforcé
classe : Prêtre
niveau requis : 5
coût : 9 PM
soin : 16-28 PV
cible : 1 allié
```

Nouveau sort découvert dans le donjon :

```text
Soin de groupe
coût : 9 PM
soin : 7-13 PV
cible : toute l'équipe
découverte : spell_group_heal
emplacement : étage 2, x 21, y 8
```

Le Soin de groupe ne demande pas de sélection de cible. Il s'applique directement à toute l'équipe.

## Grimoire et sorts actifs

Le grimoire hors combat permet maintenant de préparer les sorts actifs.

Comportement :

```text
- préparer un sort ne coûte pas de PM ;
- un Mage peut préparer son sort offensif actif ;
- un Prêtre peut préparer son sort de soin actif ;
- ces choix sont sauvegardés ;
- au début d'un combat, les sorts actifs sont initialisés depuis ces choix ;
- le grimoire de combat peut encore changer temporairement le sort actif ;
- changer de sort dans le grimoire de combat consomme le tour ;
- ce changement temporaire n'écrase pas le choix hors combat sauvegardé.
```

## Sauvegarde

Le format de sauvegarde passe en version 7.

Nouveau champ :

```text
active_ability_ids_by_party_slot
```

Ce champ mémorise les sorts actifs préparés par emplacement de héros.

Le champ de `v0.12` reste utilisé :

```text
discovered_ability_ids
```

## Statuts

Nouveau fichier :

```text
scripts/combat/CombatStatusEffectResolver.gd
```

Ce fichier pose la première base d'un système de statuts temporaires réutilisable.

Pour `v0.13`, il gère uniquement le poison appliqué aux monstres. La structure pourra servir plus tard pour des monstres empoisonneurs ou d'autres effets temporaires.
