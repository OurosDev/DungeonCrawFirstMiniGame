# v0.11 — Cadres UI NineSlice et correction Prêtre

Date de préparation : 2026-06-19

## Objectif

`v0.11` améliore l'identité visuelle de l'interface sans changer la boucle de gameplay. La version introduit une première texture de cadre NineSlice sombre, appliquée aux panneaux principaux, aux menus et aux boutons visibles.

Cette version inclut aussi un hotfix de cohérence : la classe d'Eldric est désormais nommée `Prêtre`, et non plus `Prêtresse`.

## Ajouts principaux

### Cadres UI NineSlice

- Ajout de l'asset :

```text
assets/ui/frames/texture_cadre_ui.png
```

- Ajout d'un helper centralisé pour créer les styles texturés :

```text
scripts/ui/theme/UIFrameStyle.gd
```

- Utilisation de `StyleBoxTexture` pour les cadres principaux.
- Marges NineSlice des grands panneaux : `16 / 16 / 16 / 16`.
- Marges réduites pour les boutons : `8 / 8 / 8 / 8`, afin de préserver leur lisibilité.

### Interface principale

La texture est appliquée aux zones principales de l'interface :

```text
cadres des héros
viewport central
cadre ennemi
journal
automap
panneaux principaux
boutons d'exploration
boutons de combat
```

Le long cadre de fond derrière les boutons de déplacement et de combat a été retiré, car les boutons ont maintenant leur propre cadre texturé.

### Menus

La texture est également appliquée aux cadres et boutons des menus via `MenuUIFactory.gd`.

Écrans concernés notamment :

```text
menu principal en jeu
inventaire
boutique
statut
équipement
grimoire hors combat
grimoire de combat
options
```

### Correction Prêtre

La classe d'Eldric est corrigée :

```text
ancien libellé : Prêtresse
nouveau libellé : Prêtre
```

La correction concerne :

```text
liste des classes
données de classe
création de groupe
progression de niveau
restrictions d'équipement
sauvegarde / chargement
```

Une normalisation centralisée permet de convertir les anciennes sauvegardes ou anciennes valeurs de classe vers `Prêtre`.

## Notes de design

Cette version sert surtout à poser la première vraie direction visuelle de l'UI : dark fantasy rétro, bronze sombre, pierre usée, cadre lisible, sans halo blanc.

Les boutons utilisent provisoirement la même texture que les panneaux. Ils restent lisibles et utilisables, mais une texture dédiée aux boutons pourra améliorer le rendu plus tard.

Le choix `Prêtre` aligne le libellé de classe avec le portrait masculin d'Eldric et évite les incohérences dans les menus, la création de groupe et l'équipement.

## Compatibilité sauvegarde

Cette version ne change pas volontairement le format de sauvegarde.

Le champ `job` des héros est normalisé au chargement et à la sauvegarde afin que les anciennes valeurs liées à l'ancien libellé soient converties vers `Prêtre`.

Les anciennes sauvegardes devraient donc continuer à charger, avec le libellé corrigé.
